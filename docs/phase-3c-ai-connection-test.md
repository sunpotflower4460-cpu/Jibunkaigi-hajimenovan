# Phase 3C: AI Connection Test

## 目的

設定画面から、Gemini API接続状態を手動確認できるようにする。

## 方針

- APIキーはコードに入れない
- 設定画面から `/api/gemini` を短く呼ぶ
- 未設定なら `GEMINI_API_KEY_MISSING` として表示する
- 接続失敗時もアプリ本体は壊さない
- 未設定時はローカル応答へfallbackする前提を明記する

## 変更内容

- `SettingsPanel` にAI接続確認ボタンを追加
- 接続状態を以下のように表示
  - 未確認
  - 確認中
  - 接続OK
  - 未設定
  - 要確認
- Gemini API未設定時に、設定画面上で分かるようにする

## 手動確認

- 設定 → AI接続を確認する が押せる
- `GEMINI_API_KEY` 未設定時に未設定表示になる
- APIキー設定後に接続OKになる
- 通信失敗時に要確認になる
- 設定パネルを閉じてもアプリ本体が壊れない
