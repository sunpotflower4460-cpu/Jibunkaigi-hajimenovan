# Phase 2C-20: Storage / Intro Stability

## 目的

App Store申請前に、保存まわりと旧データ耐性を強める。

## 修正内容

- 付箋ストアの保存処理を安定化
  - 保存前に必ずサニタイズ
  - 保存後に再読込した安全な配列を返す
  - 保存失敗時もサニタイズ済み配列を返し、UIが壊れにくいようにする
  - 空の sessionId / content では新規付箋を作らない
- 会議録ストアの保存処理を安定化
  - 保存前に必ずサニタイズ
  - 保存後に再読込した安全な配列を返す
  - 保存失敗時もサニタイズ済み配列を返す
  - title / topic などの最大長を補正
- 初回導線まわりの危険な direct localStorage 参照を検索確認

## 期待される効果

- 壊れた古い保存データが残っていても白画面になりにくい
- 付箋や会議録の保存失敗時にUIだけ保存済みに見えるズレが減る
- localStorage容量制限や一時的な保存失敗への耐性が少し上がる

## 手動確認項目

- 付箋を作成できる
- 付箋を削除できる
- 会議録を作成できる
- 会議録を削除できる
- リロード後も付箋/会議録が残る
- 旧データがあっても画面が白くならない
