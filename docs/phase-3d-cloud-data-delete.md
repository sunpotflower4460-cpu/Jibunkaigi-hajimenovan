# Phase 3D: Cloud Data Delete

## 目的

App Store申請を見据えて、Firebaseクラウド保存を使う場合にクラウド側データを削除できる導線を用意する。

## 方針

- 本番Firebase設定やFirestore Rulesは変更しない
- APIキーやsecretは扱わない
- 設定画面から削除できるUIを追加する
- 削除後に自動で再アップロードされないよう、クラウド同期を停止する
- 端末内データ削除とは分ける

## 変更内容

- `firebaseCloud.ts`
  - クラウド同期停止フラグを追加
  - queued cloud saveをキャンセルできるようにする
  - Firestore上の `users/{uid}/snapshots/app-state` を削除する関数を追加
  - 可能な場合は匿名Firebaseユーザーも削除する
  - 削除後はクラウド同期を停止する
  - 同期再開関数を追加
- `SettingsPanel.tsx`
  - クラウドデータ削除セクションを追加
  - 確認チェック付きの削除ボタンを追加
  - 同期再開ボタンを追加
  - 削除中 / 削除済み / 再開中 / エラー表示を追加

## 手動確認

- Firebase未設定時でも削除ボタンでアプリが壊れない
- Firebase設定済み時にクラウド削除が実行できる
- 削除後にクラウド保存状態が端末内保存へ戻る
- 削除後にすぐ再アップロードされない
- 同期再開ボタンでクラウド同期を再開できる
- 端末内の会話データはこの操作だけでは消えない

## 注意

Firebase Authの匿名ユーザー削除は、認証状態によって再ログインが必要になる場合がある。その場合でもFirestore上の保存データ削除を優先し、同期停止状態にする。
