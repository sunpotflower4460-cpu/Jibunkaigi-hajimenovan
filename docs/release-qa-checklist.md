# じぶん会議 はじめの版 - リリース前QAチェックリスト

## 目的

App Store申請前に、最低限ここだけは実機で確認するためのチェックリスト。

この版の目的は「深層版」ではなく、はじめの版として安全に触れる状態にすること。

## 確認環境

- iPhone実機
- Safari / Capacitor WebView 想定
- できれば通常回線とWi-Fiの両方
- できれば一度リロード、またはアプリ再起動後も確認

---

## 1. 初回画面

- [ ] 初回画面が白画面にならず表示される
- [ ] タイトルと世界観が崩れていない
- [ ] 大きすぎる注意カードが前面に出て雰囲気を壊していない
- [ ] 「潜る」ボタンが初回画面には出ていない
- [ ] 利用規約 / プライバシーポリシー / 注意事項を確認できる導線がある
- [ ] 同意チェックなしでは開始できない
- [ ] 同意チェック後、「会議をはじめる」で進める
- [ ] 開始後、次回起動時に毎回初回へ戻らない

---

## 2. 基本会話

- [ ] 入力欄に文字を入れられる
- [ ] 送信できる
- [ ] 送信後、ユーザーの発言が表示される
- [ ] エージェント選択ボタンが出る
- [ ] レイを選ぶと返答が出る
- [ ] ジョーを選ぶと返答が出る
- [ ] ケンを選ぶと返答が出る
- [ ] ミナを選ぶと返答が出る
- [ ] サトウを選ぶと返答が出る
- [ ] 心の鏡を選ぶと総括が出る
- [ ] 連続で数回会話しても固まらない
- [ ] 生成中表示が出たまま戻らなくならない

---

## 3. OTHERS

- [ ] 通常エージェントの返答後に OTHERS が表示される
- [ ] OTHERS は自分以外の4人だけが出る
- [ ] 心の鏡には OTHERS が出ない
- [ ] 選択中エージェント本人が OTHERS に混ざらない
- [ ] master / 心の鏡が OTHERS に混ざらない
- [ ] 各コメントが極端に同じ文ばかりになっていない
- [ ] OTHERSを開閉しても画面が崩れない

---

## 4. 画面スクロール / スマホ挙動

- [ ] 会話が長くなっても下までスクロールできる
- [ ] 入力欄がキーボードに隠れすぎない
- [ ] モーダル表示中に背景スクロールで大きく崩れない
- [ ] 各パネルの中身を下までスクロールできる
- [ ] 小さい画面でもボタンが押せる
- [ ] 右下のUIが入力欄を邪魔しすぎない

---

## 5. 潜るメニュー

- [ ] 会議開始後に「潜る」ボタンが表示される
- [ ] 潜るメニューを開ける
- [ ] 潜るメニューを閉じられる
- [ ] 潜る → 設定 が開く
- [ ] 潜る → 会議録 が開く
- [ ] 潜る → 言葉 が開く
- [ ] 潜る → 付箋 が開く
- [ ] 潜る → 輪郭 が開く
- [ ] 各パネルを閉じられる
- [ ] 旧右側フローティングボタン群が重複して見えない

---

## 6. 言葉の水面

- [ ] 言葉の水面が開く
- [ ] 長いフレーズが枠外へ突き抜けない
- [ ] 「言葉」「胸」「違和感」など短い語として表示される
- [ ] 語同士が重なりすぎない
- [ ] 文字が縦長につぶれない
- [ ] 更新ボタンを押しても壊れない
- [ ] 「この水面にどう思う？を貼る」で付箋が開く

---

## 7. 付箋

- [ ] 付箋パネルが開く
- [ ] 付箋を書ける
- [ ] 付箋を保存できる
- [ ] 保存後に一覧へ出る
- [ ] 付箋を削除できる
- [ ] リロード後も付箋が残る
- [ ] 空欄では保存されない

---

## 8. 会議録

- [ ] 会議録パネルが開く
- [ ] 会話後に会議録を作成できる
- [ ] 会議録が一覧に出る
- [ ] 会議録を削除できる
- [ ] リロード後も会議録が残る
- [ ] 会議録から「どう思う？」付箋へ進める

---

## 9. 輪郭

- [ ] 輪郭パネルが開く
- [ ] 会話 / 会議録 / 付箋の蓄積があるとテーマが表示される
- [ ] 何もない場合も白画面にならない
- [ ] 輪郭から「どう思う？」付箋へ進める
- [ ] 更新ボタンで壊れない

---

## 10. 保存

- [ ] 会話を作ったあとリロードしても残る
- [ ] 新しい問いを作っても過去の問いが残る
- [ ] 名前変更が保存される
- [ ] 付箋が保存される
- [ ] 会議録が保存される
- [ ] 保存データが多くなっても極端に重くならない

---

## 11. 設定

- [ ] 設定パネルが開く
- [ ] 保存状態が表示される
- [ ] 端末内保存の説明が表示される
- [ ] AI API接続 / 広告 / 課金の項目が誤解を招きすぎない
- [ ] ご利用についての注意が見られる
- [ ] 設定パネルを閉じられる

---

## 12. リリース前にNGなら止める条件

以下が1つでもある場合は、App Store申請前に止める。

- [ ] 初回画面から進めない
- [ ] 会話送信で固まる
- [ ] 返答生成中のまま戻らない
- [ ] 画面が白くなる
- [ ] スクロールできず主要機能に届かない
- [ ] 保存がまったく効かない
- [ ] 利用規約 / プライバシー / 注意事項への導線がない
- [ ] 緊急対応や医療代替に見える文言が前面に出ている
- [ ] APIキーやsecretが画面やコードに露出している

---

## 13. App Store申請前の判断

### 申請してよい状態

- 基本会話が安定している
- 保存が最低限効いている
- 初回同意導線がある
- 注意事項が確認できる
- 主要パネルが開閉できる
- スマホで大きく崩れない
- 開発途中感のある文言が前面に出ていない

### まだ申請しない状態

- 生成AI本接続を入れる予定が残っている
- 規約/プライバシーポリシーが未確定
- Capacitorビルドで表示確認できていない
- iPhone実機でスクロール/入力に問題がある
