# Phase 2C-19: Release Bug Sweep

## 目的

App Store申請前に、直近で追加した導線やパネル周りの安定性を確認し、体験を壊しやすいバグを減らす。

## 修正内容

- 開始画面では Dive Tools Dock / 潜るボタンを非表示にする
- Dive Tools Dock から各パネルを開く処理を、隠しボタンクリック依存からイベント方式へ変更する
- 設定 / 会議録 / 言葉の水面 / 付箋 / 輪郭パネルを `subscribeDiveTool` に対応させる
- 旧フローティングボタンが後から描画されても `data-legacy-dive-tool` で隠れるようにする
- 会議録と言葉の水面から、制作途中感のある文言を取り除く

## 期待される効果

- 開始前の雰囲気が壊れにくくなる
- 潜るメニューの各項目が安定して開く
- 隠しボタンに依存した二重起動や表示崩れのリスクが下がる
- リリース前の見え方として自然になる

## 手動確認項目

- 初回同意画面で「潜る」ボタンが出ない
- 会議開始後に「潜る」ボタンが出る
- 潜る → 設定 が開く
- 潜る → 会議録 が開く
- 潜る → 言葉 が開く
- 潜る → 付箋 が開く
- 潜る → 輪郭 が開く
- 各パネルが閉じられる
- スクロールが詰まらない
- 古い右側フローティングボタン群が重複表示されない
