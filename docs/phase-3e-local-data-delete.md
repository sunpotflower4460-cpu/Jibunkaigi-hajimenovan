# Phase 3E: Local Data Delete

## 目的

App Store申請を見据えて、ユーザーがこの端末に保存されたデータを削除できる導線を用意する。

## 方針

- 端末内データ削除とクラウドデータ削除を分ける
- 端末内削除は localStorage / IndexedDB / 付箋 / 会議録を対象にする
- 誤操作防止のため確認チェックを必須にする
- 削除後は画面状態を確実にリセットするため再読み込みする

## 変更内容

- `stickyNoteStore.ts`
  - `clearStickyNotes()` を追加
- `conferenceRecordStore.ts`
  - `clearConferenceRecords()` を追加
- `SettingsPanel.tsx`
  - 端末内データ削除セクションを追加
  - 確認チェック付きの削除ボタンを追加
  - 削除中 / 削除済み / エラー表示を追加
  - 削除後にページを再読み込みする

## 削除対象

- 会話セッション
- 会話メッセージ
- ユーザー名 / 同意状態などの端末内設定
- 付箋
- 会議録

## 削除対象外

- Firebaseクラウド側の保存データ
- Gemini APIキーなどのサーバー側環境変数
- App Store / Vercel / Firebase側の設定

## 手動確認

- 設定 → 端末内データ削除 が表示される
- チェックなしでは削除ボタンを押せない
- チェック後に端末内データを削除できる
- 削除後に画面が再読み込みされる
- 再読み込み後、会話・付箋・会議録が消えている
- クラウドデータ削除とは別導線として見える
