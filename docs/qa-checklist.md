# じぶん会議 はじめの版 QAチェックリスト

App Store初期版へ進める前に、Vercel Preview・スマホ実機・ブラウザで確認する項目です。

## 1. 初回起動・同意フロー

- [ ] 初回表示でイントロ画面が出る
- [ ] 利用規約チェックなしでは「会議をはじめる」ボタンが押せない
- [ ] チェック後に「会議をはじめる」ボタンが押せる
- [ ] 同意後は通常画面に入れる
- [ ] 再起動後は同意画面が再表示されない
- [ ] localStorageリセット後は同意画面が再表示される

## 2. 会話

- [ ] 空入力では送信できない
- [ ] 新規セッション作成して1行入力を送信できる
- [ ] 長文入力を送信できる
- [ ] 送信後に入力欄が閉じ、エージェント選択へ進む
- [ ] 5エージェント（レイ・ジョー・ケン・ミナ・サトウ）がそれぞれ応答する
- [ ] 委ねるがランダムに応答する
- [ ] 心の鏡が応答する
- [ ] OTHERSが表示・展開できる
- [ ] メッセージ削除できる
- [ ] メッセージをコピーできる

## 3. 生成中の操作

- [ ] 応答生成中はセッション切替できない
- [ ] 応答生成中は削除できない
- [ ] 応答生成中はモード変更できない
- [ ] 生成完了後に操作可能へ戻る
- [ ] 応答が別セッションへ誤保存されない

## 4. セッション管理

- [ ] 新しい問いを作れる
- [ ] セッション一覧に表示される
- [ ] セッションタイトルを編集できる（セッション選択と誤爆しない）
- [ ] ピン留めできる（セッション選択と誤爆しない）
- [ ] セッション削除確認モーダルが出る
- [ ] セッション削除できる
- [ ] 現在開いているセッション削除後に安全にホームへ戻る

## 5. 保存・復元

- [ ] リロード後に会話が残る
- [ ] localStorage破損時にアプリが真っ白にならない（ErrorBoundaryで復帰できる）
- [ ] IndexedDB復元が効く（IndexedDB側が新しければ起動時に復元される）
- [ ] 端末内の保存データリセット後に初回画面へ戻る

## 6. メッセージ操作

- [ ] メッセージをタップしてツールバーを表示できる
- [ ] コピーできる
- [ ] メッセージ削除できる
- [ ] OTHERS展開中に削除しても表示が崩れない

## 7. 名前設定

- [ ] 名前変更モーダルを開ける
- [ ] 名前を保存できる
- [ ] 空文字は保存されない
- [ ] 30文字を超えても崩れない

## 8. スマホ表示

- [ ] iPhone SafariでUI全体が表示崩れしない
- [ ] サイドバーを開閉できる（誤爆しない）
- [ ] 横スクロールのエージェントボタンが押しやすい
- [ ] 入力欄がキーボード表示で大きく崩れない
- [ ] 下部safe-areaに文字やボタンが隠れない
- [ ] 長文・長い単語が画面外にはみ出さない
- [ ] モーダルが画面内でスクロールできる
- [ ] セッション削除確認モーダルが押しやすい
- [ ] OTHERS展開が長くなってもスクロールできる

## 9. 法務表示

- [ ] 利用規約を開ける
- [ ] プライバシーポリシーを開ける
- [ ] 緊急時案内（ご利用について）を確認できる
- [ ] 医療・診断ではない旨が表示される
- [ ] mock AIであることが「この版について」に明記されている
- [ ] 利用規約の最終更新日が最新になっている
- [ ] プライバシーポリシーの最終更新日が最新になっている

## 10. App Store向け確認

- [ ] 「ご利用について」から注意文を確認できる
- [ ] mock AIであることをREADMEで把握できる
- [ ] secret / API key / token がコードに含まれていない
- [ ] Firebase / Gemini / 認証 / 課金に未接続であることが明記されている

## 11. Vercel Preview確認

- [ ] Vercel Preview URLが開ける
- [ ] iPhone Safariで表示できる
- [ ] PC幅でも表示できる
- [ ] ブラウザリロード後も履歴が残る
- [ ] VercelのBuild Logsにエラーがない

## 12. 次Phaseへ進む条件

- [ ] `npm run typecheck` が成功している
- [ ] `npm run build` が成功している
- [ ] CIが成功している
- [ ] Vercel Previewが表示できる
- [ ] スマホ操作で明らかな崩れがない
- [ ] 重大なクラッシュがない
- [ ] mock版として一通り触れる

ここまで確認できたら、Phase 2でGemini実接続準備へ進めます。
